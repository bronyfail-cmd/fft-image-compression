
## 0. Тестовое изображение
`generate_test_image()` создаёт картинку 256×256 без файла: шахматка (резкие границы → высокие частоты), светлый круг (плавный → низкие частоты), пара прямых линий и шум. Так на одном изображении видно, как спектр «теряет» и резкие, и гладкие детали при сжатии.

## 1-4. Основная функция сжатия — `compress_by_spectrum_truncation`

**Шаг 1 (БПФ):**
```python
F = np.fft.fft2(img_f)
```
`fft2` — двумерное Фурье-преобразование. Получаем комплексную матрицу той же размерности — каждый элемент `F[u][v]` это амплитуда+фаза определённой пространственной частоты.

**Шаг 2 (сортировка по амплитуде):**
```python
magnitude = np.abs(F)              # модуль комплексного числа = амплитуда
flat_magnitude = magnitude.flatten()  # 2D -> 1D, чтобы было удобно сортировать
sorted_indices = np.argsort(flat_magnitude)[::-1]  # индексы по убыванию
```

`argsort` возвращает не сами значения, а индексы, которые бы расставили массив по возрастанию. `[::-1]` разворачивает — получаем индексы от самой большой амплитуды к самой маленькой.

**Шаг 3 (оставить top-N%, остальное обнулить):**
```python
n_keep = max(1, int(round(n_total * percent / 100.0)))
indices_to_keep = sorted_indices[:n_keep]

mask_flat = np.zeros(n_total, dtype=bool)
mask_flat[indices_to_keep] = True
mask = mask_flat.reshape(n_rows, n_cols)

F_truncated = F * mask
```
Считаем, сколько коэффициентов это N% от общего числа, берём столько самых "сильных" индексов. Создаём булеву маску (True там, где коэффициент оставляем), разворачиваем её обратно в форму изображения и умножаем на спектр — `False` (т.е. 0) умножает остальные коэффициенты в ноль.

**Шаг 4 (обратное БПФ):**
```python
img_reconstructed = np.fft.ifft2(F_truncated)
img_reconstructed = np.real(img_reconstructed)
img_reconstructed = np.clip(img_reconstructed, 0, 255).astype(np.uint8)
```

`ifft2` восстанавливает изображение из урезанного спектра. Результат теоретически должен быть вещественным (раз исходная картинка вещественная), но из-за погрешностей вычислений остаётся крошечная мнимая часть — берём `np.real`, отбрасывая её. `clip` подрезает выход в диапазон [0,255], так как после обнуления части спектра могут появиться значения чуть за пределами.

## 5. Метрики — MSE и PSNR
```python
def compute_mse(original, reconstructed):
    return np.mean((original - reconstructed) ** 2)

def compute_psnr(original, reconstructed, max_pixel=255.0):
    mse = compute_mse(original, reconstructed)
    if mse == 0:
        return float('inf')
    return 20 * np.log10(max_pixel / np.sqrt(mse))
```
MSE — средний квадрат разницы пикселей. PSNR — стандартная формула из задания, переводит MSE в децибелы; чем больше PSNR, тем меньше искажение.

## Дополнительное задание — сохранение в виде индексов
```python
def save_compressed(F_truncated, mask, shape, filepath):
    rows, cols = np.nonzero(mask)
    values = F_truncated[rows, cols]
    data = {"shape": shape, "rows": rows, "cols": cols, "values": values}
    pickle.dump(data, open(filepath, "wb"))
```
Вместо хранения всей матрицы спектра (256×256 комплексных чисел) сохраняем только координаты (`rows`, `cols`) ненулевых коэффициентов и их значения — это и есть «сжатое представление». Дальше считается выигрыш в размере (`full_bytes / sparse_bytes`).
