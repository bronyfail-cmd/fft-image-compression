## Общая идея
Алгоритм сжимает изображение, обнуляя "слабые" коэффициенты Фурье-спектра и оставляя только N% самых сильных по амплитуде. Логика: в спектре большая часть энергии изображения сосредоточена в небольшом числе крупных коэффициентов, поэтому можно отбросить большинство и восстановить картинку почти без потерь.

## Шаг 1 — Прямое БПФ
```python
F = np.fft.fft2(img_f)
```
`fft2` переводит изображение из пространственной области (пиксели) в частотную (спектр). Получаем комплексную матрицу той же размерности — каждое значение содержит амплитуду и фазу одной пространственной частоты.

## Шаг 2 — Сортировка по амплитуде
```python
magnitude = np.abs(F)
flat_magnitude = magnitude.flatten()
sorted_indices = np.argsort(flat_magnitude)[::-1]
```

`np.abs` от комплексного числа даёт его модуль — это и есть амплитуда. `flatten` превращает 2D-матрицу в 1D-массив, чтобы можно было отсортировать все коэффициенты вместе, не привязываясь к их положению (строка/столбец). `argsort` возвращает индексы элементов в порядке возрастания значений, `[::-1]` разворачивает порядок — получаем индексы от самой большой амплитуды к самой маленькой.

## Шаг 3 — Оставляем top-N%, остальное в ноль
```python
n_keep = max(1, int(round(n_total * percent / 100.0)))
indices_to_keep = sorted_indices[:n_keep]

mask = np.zeros(...)
mask[indices_to_keep] = True
mask = mask.reshape(n_rows, n_cols)

F_truncated = F * mask
```
Считаем, сколько коэффициентов соответствует N% от их общего числа. Берём столько самых сильных индексов. Строим булеву маску (True — оставить, False — занулить) и умножаем на спектр: там, где маска False, коэффициент превращается в 0.

## Шаг 4 — Обратное БПФ
```python
img_reconstructed = np.fft.ifft2(F_truncated)
img_reconstructed = np.real(img_reconstructed)
img_reconstructed = np.clip(img_reconstructed, 0, 255).astype(np.uint8)
```
`ifft2` возвращает изображение обратно в пространственную область из урезанного спектра. Так как часть коэффициентов выкинута, результат уже не идентичен оригиналу — это и есть потеря качества от сжатия. Берём `np.real`, потому что из-за погрешностей вычислений остаётся ничтожно малая мнимая часть. `clip` подрезает выход в диапазон 0-255, так как после обнуления коэффициентов значения пикселей могут чуть выйти за границы.

## Шаг 5 — Метрики качества: MSE и PSNR
```python
mse = np.mean((original - reconstructed) ** 2)
psnr = 20 * np.log10(255 / sqrt(mse))
```
MSE — средний квадрат разницы между пикселями оригинала и восстановленного изображения. PSNR переводит эту ошибку в децибелы по стандартной формуле; чем выше PSNR, тем меньше искажение.

## Результаты эксперимента
Прогон для N = 50%, 10%, 5%, 1% показал ожидаемую закономерность: чем меньше процент оставленных коэффициентов, тем больше MSE и ниже PSNR (с 37.45 дБ при 50% до 22.42 дБ при 1%) — то есть качество падает плавно и предсказуемо, без скачков. Это подтверждает, что метод работает корректно: даже сохранив всего 10% коэффициентов, мы получаем PSNR ~29.5 дБ, что визуально вполне приемлемо.

## График PSNR(N%) и MSE(N%)
Строится по более широкому диапазону процентов (от 0.5% до 100%), показывает, что зависимость нелинейная: на первых процентах PSNR растёт очень быстро (потому что там сосредоточена основная энергия изображения), а дальше прирост замедляется — классическая картина для Фурье-сжатия.

## Дополнительное задание — компактное хранение
```python
rows, cols = np.nonzero(mask)
values = F_truncated[rows, cols]
pickle.dump({"shape": ..., "rows": rows, "cols": cols, "values": values}, file)
```
Вместо сохранения полной матрицы спектра (256×256 комплексных чисел = больше 1 млн байт) сохраняем только координаты ненулевых коэффициентов и их значения — получили сжатое представление в 209 728 байт, то есть в 5 раз меньше при N=10%. При восстановлении создаём пустую матрицу нужного размера и расставляем сохранённые значения по сохранённым координатам, затем делаем обратное БПФ — и получаем то же самое изображение, что и при прямом восстановлении (это подтверждено проверкой `assert` в коде).
